# 📝 AI Prompting Templates for KDP Content Creation

> Use these templates with ChatGPT or Claude to generate high-quality book content.  
> Always apply human review before publishing.

---

## Template 1 — Niche Validation Prompt

```
I want to write a non-fiction book for Amazon KDP.

Target niche : [YOUR NICHE]
Target reader: [WHO THEY ARE + their #1 struggle]

Please:
1. Confirm if this niche has clear reader demand
2. Suggest 3 more specific sub-niches I should consider
3. Identify the top 3 problems readers in this niche face
4. Suggest a working book title with primary keyword included
```

---

## Template 2 — Modular Chapter Outline

```
I am writing a [PAGE COUNT]-page non-fiction book titled:
"[YOUR BOOK TITLE]"

Target reader: [READER DESCRIPTION + experience level]
Core promise : [What the reader will achieve after reading]

Generate a complete chapter outline where:
- Each chapter solves ONE specific problem
- Each chapter title includes an action verb
- Chapters build on each other progressively
- Total chapter count is between 8 and 12

Format each chapter as:
Chapter N: [Title]
- Core problem addressed:
- Key framework or method:
- Expected reader outcome:
```

---

## Template 3 — Chapter Expansion (Recursive)

```
CONTEXT: I am writing Chapter [N] of my book "[TITLE]".
READER  : [Who they are, what they already know]
CHAPTER : [Chapter title]
PROMISE : [What the reader will know/do after this chapter]

STEP 1 — Opening Hook (150 words):
Write an opening that immediately validates the reader's
frustration with [SPECIFIC PAIN POINT]. Be direct, empathetic,
and specific. Avoid generic statements.

STEP 2 — Core Framework (300 words):
Provide a [3/4/5]-step framework to solve [PROBLEM].
Each step must:
- Have a clear, memorable name
- Be actionable within 24 hours
- Include one specific example

STEP 3 — Deep Dive on Step [N] (200 words):
Expand with:
- A real-world scenario (not hypothetical)
- The #1 mistake people make at this step
- One metric to know if you're doing it right

STEP 4 — Chapter Summary + Bridge (100 words):
Close with a 3-bullet summary of key takeaways.
End with a one-sentence bridge to the next chapter.
```

## Template 4 — Amazon Description Writer

```
Write an Amazon book description for:

Title    : [BOOK TITLE]
Subtitle : [BOOK SUBTITLE]
Niche    : [NICHE / CATEGORY]
Reader   : [TARGET READER DESCRIPTION]
Promise  : [Core transformation the book delivers]
Length   : [PAGE COUNT] pages

Requirements:
- Open with a bold question or pain point statement
- Use HTML formatting: <b> for bold, <ul><li> for bullet points
- Include 5 concrete benefits in bullet point format
- End with a clear call-to-action ("Scroll up and click Buy Now")
- Total length: 150–200 words
- Tone: [professional / conversational / motivational]
- Do NOT use the word "comprehensive" or "ultimate guide"
```

## Template 5 — Voice Refinement

```
Rewrite the following section in the voice of [AUTHOR PERSONA]:

[PASTE YOUR AI-GENERATED TEXT HERE]

Voice characteristics:
- Tone      : [direct / warm / authoritative / conversational]
- Style     : [short punchy sentences / flowing narrative]
- Avoid     : passive voice, filler phrases, overly formal language
- Include   : [first-person anecdotes / rhetorical questions / data points]

The reader is [READER DESCRIPTION].
They should feel [EMOTIONAL RESPONSE] after reading this section.
```

---

## Template 6 — Review Request (Back Matter)

```
Write a 50-word review request to include at the end of my book.

Book topic: [TOPIC]
Tone      : [warm / casual / professional]

Requirements:
- Acknowledge the reader just finished
- Make leaving a review feel easy and low-pressure
- Mention it takes less than 60 seconds
- Include one specific thing they could mention in their review
- No begging, no guilt — just a genuine ask
```

## ⚠️ Quality Control Checklist (After Every AI Draft)

Before accepting any AI-generated content, verify:

- [ ] All facts and statistics are independently verified
- [ ] No generic filler phrases ("In today's fast-paced world...")
- [ ] Voice sounds human, not robotic
- [ ] Every step is actually actionable (not vague advice)
- [ ] Examples are specific, not hypothetical
- [ ] No contradictions between chapters
- [ ] AI disclosure added to KDP upload form if applicable
